# PAPA — Found Founded HR 시스템 디자인 브리프

> Claude 디자인 세션에 그대로 투입할 수 있는 프로젝트 컨텍스트 문서

---

## 1. 프로젝트 개요

- **제품명**: PAPA (파파)
- **정체성**: Found Founded(파운드파운디드) 내부 HR / 근태 시스템
- **네이밍 근거**: 회사 규정의 "파파 유연 출퇴근 근태" 표현에서 차용
- **조직 규모**: 크리에이티브 디자인 스튜디오, 7명 내외
- **목적**: 출퇴근·휴가·결재·조직 정보를 단일 워크스페이스로 통합
- **배포 환경**: 웹 우선, 모바일 대응 필요(PWA 고려)

---

## 2. 사용자 역할 & 권한 (3단계)

| 역할 | 국문 | 주요 권한 |
|---|---|---|
| Admin | 관리자(대표) | 전체 열람, 최종 결재 |
| Senior | 시니어(리더) | 멤버 기능 + 1차 결재 + 입사일 열람 |
| Member | 멤버(직원) | 본인 정보 + 팀 현황(입사일 제외) |

**권한별 화면 차등**
- 멤버: 대시보드 / 출퇴근 / 지각 / 휴가신청 / 캘린더 / 조직도 / 근무규정
- 시니어 = 멤버 + 결재함 (멤버 신청 1차 결재)
- 관리자 = 시니어 + 최종 결재 + 시니어 신청 직접 결재

---

## 3. 기능 구조

1. **대시보드** — 역할별 위젯 조합, 오늘의 나 + 팀 현황 + 연차 잔여 + 결재 대기
2. **출퇴근** — 체크인/아웃, 점심시간 선택(1h / 1.5h), 주간 로그
3. **지각했어요** — 자진 신고 폼, 누적 카운터(5회 한계), 벌칙 상태, 전사 피드
4. **휴가 신청** — 연차/반차/리프레시 신청, 결재 상태 추적
5. **결재함** — 시니어/관리자 전용, 대기 + 처리 완료 리스트
6. **캘린더** — 휴가·반차·생일 색상 구분, 월간 그리드
7. **조직도** — 대표→시니어→팀원 3단 트리, 권한별 입사일 노출
8. **근무 규정** — 정책 문서 뷰 (유연근태 / 지각 / 점심 / 휴일수당 / 연차표)

## 4. 비즈니스 규칙 (비주얼에 반영 필수)

### 4-1. 유연 출퇴근
- 9~11시 자율 출근 (월요일은 오후 1시)
- 프로젝트 일정상 정상 출근 가능
- **출근 시간에 맞춰 퇴근**: 예) 11시 출근 → 8시 퇴근

### 4-2. 지각 벌칙 시스템
- 1분이라도 늦으면 "지각했어요 방"에 자진 신고
- 누적 5회 도달 시:
  - 1주간(5회) 10시 출근 고정
  - 2주간(2회) 월요일 10시 출근 고정
- 벌칙 시행 후 카운터 0으로 리셋
- 벌칙 기간 중 휴가 사용 시 벌칙일 증가
- **UI 요구**: 카운터 progress dots (5칸), 벌칙 중 danger 강조 카드

### 4-3. 점심시간 유연제
- 1시간 또는 1시간 30분 선택
- 1.5시간 선택 시:
  - 11시 전까지 "밥먹어요 방"에 글 남기기
  - 식사 중 변경 불가
  - 30분 연장 근무로 상쇄
- 시간대 유연: 12~1:30 또는 12:30~2:00
- 프로젝트 리딩 멤버와 사전 상의

### 4-4. 휴일 근무 수당
- 반일 (4h 30m 휴게 포함): **80,000원**
- 종일 (9h 휴게 포함): **160,000원**

### 4-5. 근속연수별 연차 (반드시 표로 시각화)

| 근속 | 총 연차 | 일반 연차 | 리프레시 | 비고 |

### 4-6. 결재 프로세스
- **멤버의 휴가 신청**: 시니어(1차) → 관리자(최종)
- **시니어의 휴가 신청**: 관리자 직행
- 결재 단계 시각화 필수 (체크마크 + 승인자명)

## 5. 디자인 시스템

### 5-1. 비주얼 디렉션
**밝고 친근한 모던 SaaS** — 시프티(Shiftee)·Lattice류. 차가운 블루 그레이 배경에 또렷한 블루 프라이머리, 라운드 코너, 소프트 섀도우, 볼드 Pretendard. 컬러풀 액센트 타일(민트/옐로우/레드)로 상태 구분.

### 5-2. 컬러 토큰

```
--bg           #F0F4FA   페이지 배경 (light blue-gray)
--surface      #FFFFFF   카드 배경
--ink          #1A2340   메인 텍스트
--ink-soft     #5A6784   보조 텍스트
--ink-mute     #9BA7C1   라벨/캡션
--line         #EAEFF7   구분선
--accent       #3A6FF0   프라이머리 블루
--accent-soft  #E4EDFF   블루 배경 소프트
--accent-dark  #2A5AD4   호버/프레스
--warn         #F5A623   주의 (옐로우)
--warn-soft    #FFF4D6   주의 배경
--ok           #3DCFA6   성공 (민트)
--ok-soft      #DCF5EE   성공 배경
--danger       #FF5A6E   위험/벌칙 (레드-코럴)
--danger-soft  #FFE4E8   위험 배경
```

### 5-3. 타이포그래피
- **폰트**: Pretendard (한글·영문 공통)
- **Display**: 700 weight, letter-spacing -0.03em
- **Heading**: 600 weight, letter-spacing -0.02em
- **Body**: 400 weight
- **Label/Caption**: 500 weight, 11px, uppercase tracking-wider
- **숫자**: font-variant-numeric: tabular-nums

### 5-4. 라운딩 & 섀도우
```
--radius-sm    8px     작은 뱃지, 버튼 내부 요소
--radius       14px    기본 카드
--radius-lg    20px    대형 히어로 카드
pill           999px   역할 뱃지, 상태 chip
--shadow       0 1px 3px rgba(26,35,64,.04), 0 4px 16px rgba(26,35,64,.04)
--shadow-lg    0 8px 32px rgba(26,35,64,.08)
```

### 5-5. 컴포넌트 패턴

- **카드**: white + soft shadow + 14px radius, 하드 보더 금지
- **프라이머리 버튼**: blue filled, 흰 텍스트, 10px radius, 호버 시 살짝 떠오름
- **고스트 버튼**: white + 1px light border, 호버 시 accent-soft 배경
- **역할 뱃지**: pill shape
  - 관리자: 다크네이비 bg, white fg
  - 시니어: blue bg, white fg
  - 멤버: accent-soft bg, blue fg
- **아바타**: 원형(50%), 컬러 배경 + 이니셜
- **인풋**: `#F5F8FC` 배경, light border, 10px radius
- **진행 bar**: 999px 라운딩, 소프트 그레이 배경 + accent fill

### 5-6. 레이아웃
- 사이드바: 240px 고정, white 배경, 활성 항목은 accent-soft 배경 + accent 텍스트
- 탑바: 64px 높이, 날짜 + 역할 전환 드롭다운
- 콘텐츠 최대폭: 1152px (max-w-6xl), 좌우 40px 패딩

## 6. 핵심 화면별 비주얼 강조점

- **대시보드 · 오늘의 나**: 블루 그라디언트 배경 카드 (linear-gradient 135deg, #3A6FF0 → #5B8BF5), 흰 타이포, 흰 퇴근 버튼
- **지각 카운터**: 5칸 progress dots, 3회 이상 시 accent 색상 경고
- **벌칙 중 상태 카드**: danger red 배경 + 흰 텍스트 + Flame 아이콘
- **양호 상태 카드**: 민트 소프트 배경 원형 안에 체크 아이콘
- **연차 잔여**: 큰 숫자 + 진행률 bar (accent fill)
- **캘린더 이벤트 태그**:
  - 연차: ok-soft 배경 + ok 텍스트
  - 반차: warn-soft + warn
  - 생일: accent-soft + accent
  - 오늘: accent 도트 마커
- **결재 카드**: 아바타 + 정보 그리드 3열(유형/기간/신청일) + 승인/반려 버튼 세트
- **조직도**: 3단 트리, 연결선(1px solid --line), 멤버에게는 입사일 가림

## 8. 디자인 톤 앤 매너 지침

- **긍정적·친근함**: 동료처럼 느껴지는 UX 라이팅 ("안녕하세요, 규호님")
- **명확한 정보 위계**: Eyebrow(작은 대문자 라벨) → Heading → Body
- **과장되지 않은 모션**: fadeIn 0.3s, 호버 시 버튼 살짝 떠오름 정도
- **서체는 본문 외에는 굵게**: 한글 Pretendard Bold가 주력
- **이모지는 한정적**: 밥먹어요 안내 📌, 생일 🎂 정도만
- **NO 다크모드** (1차 버전)
- **NO 세리프 이탤릭** (초기 버전에 있던 editorial 스타일은 제거됨)
